# PostgreSQLデバッグ体験ハンズオン

PostgreSQL のソースコードをビルドし、意図的にバグを仕込んで `gdb` で解析するまでをハンズオンで体験してみよう、という記事です
今回はソースコードをビルドし、実行ファイルを作成して少しpostgresに触れてみる、というところまでです。次の記事でgdbをいじりデバッグをします。

# 1. 環境構築
以下のDocker環境を使います

https://github.com/Yutosaki/postgres-test

:::message
chezmoiというdotfilesの管理ツールを使用しており、ここは個人の設定ファイルに書き換えてもらって構いません
:::

docker 立ち上げ方
 ```bash
docker image build -t pg-test .
```

cloneしたdockerファイルのあるディレクトリ内で、上記コマンドを使用してイメージをビルド
その後、コンテナを作成する際に`--cap-add=SYS_PTRACE`と`--security-opt seccomp=unconfined`というオプションを付けます。
基本的にDockerでは、コンテナがホストのOSを乗っ取らないようにプロセスなどの機能に制限を掛けています。しかし今回は、デバッグをしたいのでそれらの機能を一部使える様にします
- SYS_PTRACE: プロセスが別プロセスのメモリを覗いたり一時停止させる機能を許可します
- seccomp=unconfined: Dockerがデフォルトで禁止している、危険な可能性のあるシステムコールを全て許可するというものです。(`ptrace`など。これを使用して、悪意のあるプログラムが他のプロセスのパスワードを見たり、挙動を書き換えたりできてしまうため、デフォルトでは禁止されている)
```bash
docker container run -it --name pg-test --cap-add=SYS_PTRACE --security-opt seccomp=unconfined pg-test
```

## 2. ビルド・インストール手順
では、コンテナに入って作業を進めていきましょう

まずはpsotgresのソースコードをダウンロードします

```bash
git clone https://github.com/postgres/postgres
```

### ソースの構成（configure）

デバッグシンボルを有効にし、最適化を無効化した状態で `Makefile` を作成します。

```bash
./configure --prefix=$HOME/pgsql-debug --enable-debug --enable-cassert CFLAGS="-O0 -g"
```

### ビルド（並列実行）

`-j`オプションを付けることで指定したコア数で並列にmakeをしてくれます

https://qiita.com/ymdymd/items/312c9f554d4ffb1f8dc6

`nproc`でマシンの論理プロセッサ数を取得できます

```bash
make -j$(nproc)
```

以下のコマンドで、正しくmakeができたかテストできます
```bash
make check
```
### インストール

作成した実行ファイルなどを指定した場所に置くことができます
今回は`./configure`の時に指定した、$HOME/pgsql-debugに実行ファイルなどを起きます
```bash
make install
```

## 3. 環境変数設定

### 実行パスの追加

どこからでも自分の作成した実行ファイルが叩けるようにpathを通します
```bash
export PATH="$HOME/pgsql-debug/bin:$PATH"
```

### データディレクトリの指定

通常、postgreを起動するときはデータ(データの本体以外にもWALや設定ファイルなど全ての情報)をどこに保存するかを毎回指定しなければなりません。しかし、以下のPGDATAを設定しておくとpostgreの各コマンドが自動で判断してくれます。設定しない場合は`-D`オプションで`pg_ctl start -D /home/pgdata`などと毎回指定しなければなりません

```bash
export PGDATA="$HOME/pgdata"
```

### 反映確認

```bash
which postgres
```

---

## 4. DB の初期化と起動

### データベースの初期化

データベースの初期化をします。このコマンドにより、OS常に設定ファイルやテンプレートDBなどが作成されます
```bash
initdb
```

この場合は先ほどPGDATAを設定したという想定のコマンドですが、設定していない場合は
```bash
initdb -D $HOME/pgdata
```
などとする必要があります。また、どこに保存するか、どのファイルを消すか、などを明確にするために`-D`オプションで毎回指定する人もいます

### サーバの起動

次にpostgresという親プロセスをバックグラウンドで起動します。これによりDBの作成などが可能になります。
ここでは共有メモリの確保などをし、指定したlogfileに起動時のメッセージやエラーを書き込みます

```bash
pg_ctl start -l logfile
```

### 動作確認（DB 作成と接続）

そして、以下のコマンドでtestdbという新しいDBを作成します。
これは実際には、SQLの`CREATE DATABASE testdb;`を実行しているだけです。initdbで作成されたテンプレートDBをコピーして、新しいデータベースフォルダを作成します

```bash
createdb testdb
```

その後以下の`psql`コマンドでクライアントはサーバーに対して接続要求を送ります

今回は指定なしで接続しますが、`-U`で接続するユーザー名、`-d`で接続先のデータベース名などの指定もできます
```bash
psql testdb
```

では実際にpostgresに触れてみましょう
まずは簡易的なテーブルを作成し、その後GDBでその挙動を追ってみましょう

## 5. Postgers を触ってみる

まずは GDB で中身を覗くための元となるデータを作成します。

今回は以下に従って作成します
https://www.postgresql.jp/docs/9.4/dml-insert.html

```sql
-- psql testdb の中で実行
CREATE TABLE products (
    product_no integer,
    name text,
    price numeric
);
```
上記コマンドでテーブル作成後、データを挿入

```sql
INSERT INTO products (product_no, name, price) VALUES
    (1, 'Cheese', 9.99),
    (2, 'Bread', 1.99),
    (3, 'Milk', 2.99);
```

これにてテーブルとデータ挿入が完了したので、データの取得などを試せます
postgreではメタコマンドと呼ばれる`\`から始まるコマンドと、`select`などをするsql文があるので詳しくは各自で調べてください
以下は一例です

```sql
\dt --テーブルの一覧を表示する
```

```sql
select * from products;
```

これにて、postgresを触ることができました。自分でテーブルを作成したりをデータの取得ができる様になりました。次の記事でgdbの使い方、およびデバッグの仕方を解説していこうと思います
